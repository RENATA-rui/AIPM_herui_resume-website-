⌄
S 情境
业务团队刚组建一年尚无独立官网，销售急需高质量海外线索。官网面向 B 端企业（房地产、医院、体育场馆等）提供支付/通行/考勤服务，团队需要一个能承接流量并转化线索的广告页。
T 任务
0-1 牵头搭建官网，针对不同客户设计用户旅程与分层 CTA，提升留资效率与质量。核心 AI 功能——导师带着 0-1 搭建知识库 + 混元大模型驱动的智能客服模块，架构为意图识别 + 轻量 Agent 调度 + RAG 增强检索，目标：10 条对话内找到答案。
A 行动
数据层：负责数据库数据清洗、文档切片、向量化入库。
AI 模块设计：RAG 检索增强生成 + 意图识别 + 记忆机制设计。
意图识别四层机制：① 明确意图 → 直接进入 RAG/Agent/功能调用；② 模糊意图（置信度 0.3-0.6）→ 主动提问 1-2 个澄清性问题，结合上下文重新识别；③ 仍模糊 → 长记忆检索 + 关键词匹配 + 兜底引导；④ 清晰后执行对应功能。
示例：用户问"刷掌支付怎么用？"——模糊点在于想问操作还是接入，客服反问"您是普通用户想了解支付操作，还是商户想了解接入流程？"
效果评测：上线前由产品/营销/测试/开发多方打分汇总；上线后每次知识库更新或 Prompt 修改自动跑离线评测。
R 结果
问题解决率 ≥85%（10 轮对话内），应答准确率 >90%，用户满意度 >4.5/5。官网成为海外线索获取核心引擎。

LearnFlow 学了么是一款基于 React Native + Expo 构建的沉浸式技能学习伴侣应用。以像素风格界面 + 游戏化机制让学习更有趣。针对学习动力衰减快、路径不清晰、碎片化难专注三大痛点，核心功能围绕痛点设计：

① AI 获取整理开源学习材料——LLM 自动抓取并清洗结构化知识点，按领域归类为可学习的技能树节点。
② AI 智能生成学习规划——根据用户身份（学生/职场）、兴趣和当前水平，千人千面生成个性化学习路径，自动拆分阶段与每日节点。
③ 游戏化学习引导——数字怪兽养成 + 迷你游戏恢复体力 + 番茄钟专注，通过游戏化反馈驱动持续学习闭环。

项目由 AI Coding 驱动独立全栈开发（移动端 + 后端），涵盖用户认证、AI 技能树生成、数字宠物养成、游戏化激励、Pro 会员等完整功能链路，15 个页面、37+ API 端点、12 张数据表。

核心功能实现

👾 怪兽 Agent 对话
采用 Function Calling 架构：用户输入 → LLM 判断意图 → 返回结构化 tool_calls → 服务端解析执行（调 DB/Service）→ 结果回传 LLM 生成自然语言回复。定义 10 个工具（创建领域/管理任务/写笔记/查状态等），System Prompt 注入实时上下文（领域列表/任务/怪兽状态），前端根据 toolResults 同步更新本地 UI。
🧠 AI 一键填充知识路径
用户输入学习主题后，LLM 按预设 JSON Schema 自动生成结构化技能树：领域 → 阶段（大标题）→ 学习节点（含名称/描述/资源链接）。前端以可编辑表单展示生成结果，用户可增删改任意节点。生成支持手动 + AI 混合模式——手动创建框架后一键调用 AI 补全剩余内容。

02 券商投研 RAG

📊 券商投研智能问答 RAG 系统

面向券商投研分析师、机构交易员、财富管理投顾的金融高价值场景 RAG 落地项目。基于 Dify 搭建，上传 10000 份结构化研报，采用混合检索 + Jina Reranker + 结构化 Prompt 工程，全流程贴合投研实际业务。

核心痛点
  • ①大模型幻觉：编造财报毛利率，合规风险极高
  • ②信息滞后：财报/政策实时更新，通用模型无覆盖
  • ③需求极细：通用检索匹配不到精准金融数据
  • ④语义缺失：无法识别"研发投入=研发费用"
技术栈
  • •Dify + 10000 份研报 · 块长 500 tokens
  • •混合检索（语义 0.7 + 关键词 0.3）
  • •Jina Reranker 精排 · Top K=5
  • •结构化 Prompt + Temperature 0.3
  • •热点缓存 + API 服务化 + 流式输出
Prompt 示例
角色：你是严谨的金融研报分析师。
面向用户：券商投研团队。
场景：用户提问研报相关问题。
规则：
① 仅基于给定研报生成，禁止编造。
② 结论必须标注来源 [Doc456]。
③ 无法回答时回复"现有研报未覆盖"。
④ 观点矛盾时标注分歧并列出各方依据。
⑤ 禁用"可能""大概"。
关键指标与设计方法
时延 ≤5s 的保证
知识库预加载至内存（检索 <200ms）+ BERT 蒸馏版意图分类（<100ms）+ 流式输出首 token <1s + 三层路由：简单→缓存<1s / 中等→RAG 2~3s / 复杂→异步处理。
权重 0.7/0.3
网格搜索验证（0.1~0.9）对比 nDCG@5 与 MRR 得出最优值，兼顾语义泛化与术语精确召回。
Reranker 规则
Jina Cross-Encoder 对 Top 20 候选块成对打分（0~1），降序取 Top 5，分数 <0.5 丢弃。解决双塔检索"语义相似但答案错误"问题。
